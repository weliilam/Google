# 快速测试指南

## 问题：点击扩展图标后侧边栏没有出来？

### 解决方案

v5.0.1 已修复此问题。现在的工作流程：

1. **点击扩展图标**
   - 会自动打开一个弹窗（popup）
   - 弹窗会自动尝试打开侧边栏
   - 如果侧边栏没有自动打开，点击弹窗中的"打开侧边栏"按钮

2. **侧边栏位置**
   - 侧边栏会出现在页面的**右侧**
   - 与 F12 Network 面板类似
   - 不会遮挡页面主内容

3. **查看请求**
   - 侧边栏打开后，会自动显示当前页面的所有请求
   - 点击任意请求查看详情
   - 点击"响应体"选项卡查看完整数据
   - 点击"翻译字段"按钮查看字段含义

```bash
# 在 Chrome 中访问
chrome://extensions/

# 启用开发者模式（右上角开关）

```

### 2. 打开测试页面

```bash
# 在浏览器中打开
test-page.html
```

### 3. 测试功能

1. **打开侧边栏**
   - 点击浏览器工具栏中的扩展图标
   - 侧边栏会在页面右侧打开

2. **发送请求**
   - 点击测试页面中的各种按钮（GET/POST/PUT/DELETE/PATCH）
   - 观察侧边栏中的请求列表

3. **查看响应体**
   - 点击侧边栏中的任意请求
   - 查看响应体数据
   - 验证数据是否完整

4. **测试字段翻译**
   - 点击"翻译字段"按钮
   - 验证字段是否正确翻译

## 常见问题

### Q1: 点击扩展图标没有任何反应？

**A**: 
1. 确认扩展已正确加载（在 chrome://extensions/ 中查看）
2. 确认当前标签页不是 chrome:// 开头的系统页面
3. 尝试刷新当前页面

### Q2: 侧边栏打开了，但看不到任何请求？

**A**:
1. 刷新当前页面（F5）
2. 在页面上执行一些操作（点击按钮、提交表单等）
3. 检查是否在正确的标签页上

### Q3: 看到请求了，但没有响应体数据？

**A**:
1. 确认当前页面有 XHR/Fetch 请求
2. 检查浏览器控制台是否有错误信息
3. 尝试刷新页面重新捕获

### Q4: 侧边栏显示不完整或样式错乱？

**A**:
1. 确认使用的是 Chrome 88+ 版本
2. 尝试关闭侧边栏后重新打开
3. 清除浏览器缓存后重试
